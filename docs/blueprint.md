# **App Name**: PreachPoint

## Core Features:

- Session & Rules Configuration: Admins can create and configure different session types (Individual, Group, Sunday Preaching), setting specific rules for fines (e.g., per-minute overage, fixed rates), maximum time limits, and criteria for voting and points.
- Participant & Group Management: System to register individual participants, create and manage groups, and assign both individuals and groups to specific sessions with their designated roles.
- Preaching Event Tracking: Interface for tracking the start and end times of each preaching event within a session, automatically calculating time overages based on pre-defined limits.
- Dynamic Fine Calculation: Automatically calculates fines for individuals or groups based on their recorded preaching duration and the fine rules configured for the specific session type.
- Interactive Voting System: Enable participants to cast votes for individuals or groups based on predefined categories within a session, supporting top 3 selections and group member individual votes.
- Incentive & Point Distribution: Processes voting results to calculate and allocate points or incentives to top-performing individuals or groups, with automatic distribution among group members for group awards.
- Fine Explanation Tool: Utilizes generative AI as a tool to automatically generate clear, context-specific explanations for any fines incurred, detailing how the fine was calculated based on session rules and recorded time overages.

## Style Guidelines:

- Primary color: A mid-tone blue (#296BCC) chosen to convey trust and focus, serving as the main accent for interactive elements and key information in a light color scheme. Background color: A very light, subtle blue-grey (#EDF1F5) providing a clean and serene canvas for content. Accent color: A vibrant cyan-blue (#26B9DD), analogous to the primary, used sparingly for calls to action and to draw attention to important feedback.
- Headline and body font: 'Inter' (sans-serif) will be used throughout the application. Its modern, neutral, and highly readable characteristics make it suitable for displaying numerical data, forms, and descriptive text without visual distraction, aligning with the app's need for clarity and function.
- Employ a set of clear, modern, and distinct line icons. Icons will be used consistently for navigation, session controls, voting actions, and to visually represent data states or categories, ensuring immediate recognition and enhancing user comprehension.
- A structured and modular grid layout will be implemented to clearly separate sections and data, providing an organized presentation. The design will be fully responsive, adapting seamlessly to various screen sizes while maintaining optimal readability and usability for forms and data tables.
- Subtle and purposeful animations will be incorporated for visual feedback during user interactions such as form submissions, voting confirmations, and data updates. These animations will be designed to enhance the user experience by providing clear indications of system activity without being distracting.
